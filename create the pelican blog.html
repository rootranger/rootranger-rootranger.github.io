How to go live with a Pelican blog

My last post went over the prerequisites necessary to get a Pelican blog up and running. So at this point I'm going to assume that you have Python, Pelican, a markdown editor and some means of interfacting with a GitHub repository installed on your machine. I didn't talk about installing the GitHub command line tool or its use, but that is definetly a viable option. I just found it a little unwieldy as I was learning and have now grown comfortable with the Mac GUI.

Creating Your Blog

Start by creating a directory where you will store all of your blog related files locally. Navigate to this directory and run the command:

pelican-quickstart

After running it you'll need to answer a few questions that will aid in the creastion of your pelicanconf.py file.

Don't worry about your answers too much, you can always edit this file later. Once this process is completed, you'll have a file that looks similar to the one below:

This is where the rubber meets the road. The changes you make to this file will have a significant impact on the operation and apperance of your blog. There are several key fields you'll want to make sure are accurate that are pretty self explanatory (AUTHOR, SITENAME, TIMEZONE, LINKS). Even if you aren't proficient in Python, all you really have to worry about is maintaining the format from the few examples that are prepackaged in the file.
